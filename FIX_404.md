# Исправление ошибки 404 для main.js

## Проблема

Браузер пытается загрузить файл с contenthash (`main.816682f12ad37f62a1ab.js`), но в dev режиме webpack генерирует файлы без хэша (`main.js`).

## Решение

### Шаг 1: Очистите кэш браузера

**Chrome/Edge:**

1. Откройте DevTools (F12)
2. Правой кнопкой на кнопку обновления страницы
3. Выберите "Очистить кэш и жесткая перезагрузка" (Empty Cache and Hard Reload)

Или:

1. Нажмите `Ctrl+Shift+Delete`
2. Выберите "Кэшированные изображения и файлы"
3. Нажмите "Очистить данные"

**Firefox:**

1. Нажмите `Ctrl+Shift+Delete`
2. Выберите "Кэш"
3. Нажмите "Очистить сейчас"

### Шаг 2: Остановите dev server

В терминале, где запущен `npm start`, нажмите `Ctrl+C`

### Шаг 3: Очистите кэш проекта

```powershell
# Остановите все процессы node
Get-Process -Name node -ErrorAction SilentlyContinue | Stop-Process -Force

# Очистите кэш
Remove-Item -Recurse -Force build -ErrorAction SilentlyContinue
Remove-Item -Recurse -Force .cache -ErrorAction SilentlyContinue
Remove-Item -Recurse -Force node_modules\.cache -ErrorAction SilentlyContinue
```

### Шаг 4: Перезапустите dev server

```powershell
npm start
```

### Шаг 5: Откройте в режиме инкогнито

Если проблема сохраняется, откройте приложение в режиме инкогнито:

- Chrome/Edge: `Ctrl+Shift+N`
- Firefox: `Ctrl+Shift+P`

Это гарантирует, что не будет использоваться кэш браузера.

## Альтернативное решение

Если проблема все еще возникает, попробуйте:

1. Закройте все вкладки с `localhost:3000`
2. Закройте браузер полностью
3. Запустите dev server заново: `npm start`
4. Откройте новую вкладку и перейдите на `http://localhost:3000`

## Проверка

После выполнения этих шагов:

- В Network tab (F12 → Network) должен загружаться файл `main.js` (без хэша)
- Статус должен быть 200 (OK)
- В консоли не должно быть ошибок 404
